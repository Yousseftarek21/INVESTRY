Settings Tab Requirements (Apple HIG & App Store Compliant)
Design the Settings tab to feel identical in quality and organization to Apple's native Settings app while maintaining the app's premium branding. The interface must be clean, accessible, and intuitive, using native controls, appropriate spacing, Dynamic Type, and SF Symbols on iOS.
General Principles
Follow the latest Apple Human Interface Guidelines.
Use native iOS components (toggles, navigation rows, pickers, segmented controls, disclosure indicators).
Avoid custom controls when native ones provide the same functionality.
Support Dynamic Type, VoiceOver, Switch Control, and accessibility labels.
Support Light Mode, Dark Mode, and Automatic Appearance.
Respect Reduce Motion, Reduce Transparency, and Increased Contrast accessibility settings.
Use clear, user-friendly language with no technical jargon.
Display loading states and confirmations for actions that affect user data.
Never require unnecessary permissions.

Apple Compliance Requirements
The Settings screen must:
Use native navigation patterns.
Avoid duplicate settings already provided by iOS.
Never prompt repeatedly for denied permissions.
Explain why any permission is needed before requesting it.
Include an in-app Delete Account option if account creation is supported.
Make Privacy Policy and Terms of Service easy to find.
Use secure authentication for sensitive actions.
Respect system appearance and accessibility preferences.
Avoid deceptive toggles, misleading wording, or dark patterns.
Clearly distinguish informational content from investment guidance.
UI Quality Requirements
The Settings page should feel as polished as Apple's own apps:
Native grouped sections with clear headers.
SF Symbols paired with each section.
Smooth navigation transitions.
Consistent spacing and typography.
Subtle haptic feedback where appropriate.
Skeleton loading states for remote data.
Proper empty and error states.
Full support for Dynamic Type and right-to-left layouts (Arabic).
Responsive layouts on all iPhone and Android screen sizes.
Goal: A Settings experience that is familiar to iOS users, equally polished on Android, and fully aligned with Apple HIG, App Store Review Guidelines, Google Material Design 3, and Google Play policies.
